    minihost Code Review¶

    Scope: C API (projects/libminihost/), audio I/O layer (projects/libminihost_audio/), nanobind bindings (src/minihost/_core.cpp), pure-Python layer (src/minihost/*.py), and tests. Findings cite path:line and describe a concrete defect or improvement.

    Status update¶

    All 14 items in Section 9 ("Recommended priorities") were addressed across follow-up work. See Section 10 at the end of this document for per-item status. The findings below are preserved as written so the rationale and code citations remain available; consult Section 10 for the resolution of any specific item.

    9. Recommended priorities¶

    Ranked by impact-to-effort, highest first:

    1. Fix audio-thread heap allocations in mh_process_double, mh_chain_process_*, and the listener trampoline (Section 2).
    2. Add concurrency, channel-mismatch, and RT-allocation tests (Section 7).
    3. Introduce ABI versioning macro + struct version fields before any new API is shipped (Section 5).
    4. Tighten mh_set_sample_rate and mh_set_processing_precision to fail loudly when the plugin rejects the new layout/rate post-prepare (Sections 2 and 3).
    5. Document the realtime-vs-other-thread contract more precisely in minihost.h:4-9 and add mh_close / AudioDevice ordering rules (Section 3).
    6. Pre-roll / trim by latency_samples in the Python render_midi path (Section 4).
    7. Deduplicate the two find_param_by_name implementations and the two vstpreset parsers (Sections 5 and 3).
    8. Stream render_midi_to_file instead of materializing the full waveform in memory (Section 6).
    9. Add a with context manager to Plugin and a real close() method (Section 5).
    10. Pin JUCE to a SHA in download_juce.py; add CMake install rules (Section 8).
    11. Replace desc.isInstrument heuristic in mh_probe or document its limits (Sections 2 and 4).

    Caveats and deferred work¶

    • #1 (listener trampoline): kept the mutex; replaced swap() with clear() + reserved capacity to eliminate the malloc path. Mutex hold is microseconds against pre-reserved capacity. A full lock-free MPSC ring buffer is a localized later change if measurements ever show audible glitches from the residual mutex.
    • #6 (ABI versioning): macros + mh_api_version() shipped at 1.0.0; per-struct struct_size fields not added (would be churn without benefit until the first additive change actually needs them).
    • #13 (install rules): ships static lib + headers; no find_package export. JUCE-as-private-dep is the blocker; documented in the install block.
    • #14 (mh_probe MIDI flags): documentation-only fix; the code path still uses isInstrument. Properly fixing it requires parsing VST3 moduleinfo.json ourselves -- new C code that wasn't in scope.
